# Verification and Uncertainty Doctrine

**Date:** 2026-05-06  
**Class:** Durable operational doctrine

## Core Invariants
1. An answer is only as trustworthy as the method used to produce it.
2. A result produced under a violated boundary is contaminated, even if parts of it are factually correct.
3. Honest incompleteness is acceptable; simulated completeness is not.
4. Uncertainty must be disclosed immediately when verification is not possible from current context.
5. Boundary obedience is part of the deliverable, not an optional extra.

## Verification-State vs Effort-State
- **Effort-state** means work was performed, energy was spent, and an answer was produced.
- **Verification-state** means the result was obtained using the permitted method and can be trusted under the stated rules.
- Effort-state does not imply verification-state.
- A polished result in effort-state is not a valid substitute for a verified result.

## Uncertainty Disclosure Standard
When exact verification is unavailable, say so plainly using blocker-first language:
- state the uncertainty
- state what it blocks
- state what is needed next
- do not fill the gap with guesses

## Anti-Pattern: Performed Verification
Performed verification is when the output imitates rigor while the underlying method has not satisfied the rule set.
Common signs:
- answer sounds complete but was sourced outside the allowed method
- confidence is substituted for evidence
- method violations are hidden until challenged
- results are presented as if verification occurred when it did not

## Anti-Pattern: Performed Curiosity
Performed curiosity is when questions are asked or research is done from a pre-decided conclusion rather than open-ended uncertainty.
Common signs:
- the answer is already decided before the question is asked
- questions are used to confirm, not discover
- curiosity is costumed as exploration while destination is fixed

## Reviewer Posture Rules
1. Check the method before trusting the answer.
2. Do not reward completeness when the boundary was violated.
3. Prefer honest gaps over dishonest polish.
4. Treat hidden uncertainty as a trust issue, not a cosmetic issue.
5. Preserve evidence of failure so the invariant can be reused.

## Canon-Worthy Principle
The system must optimize for truth under the stated boundary, not for the appearance of competence.
