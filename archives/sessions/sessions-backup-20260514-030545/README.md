# Session Logs

Canonical location: `sessions/YYYY/MM/`

## Filename pattern
`YYYY-MM-DD-<channel>-<session-key>.md`

Example:
`2026-05-01-telegram-6686948714.md`

## Canonical unit
One file per session.

## Required behavior
At the end of each session, write or finalize the corresponding session log in the canonical session-log lane.

## Minimal session log template
- Session: <session-key>
- Channel: <channel>
- Started: <timestamp>
- Ended: <timestamp>
- Objective:
- Actions:
- Decisions:
- Outputs:
- Blockers:
